# Wiki — Huellas de Paz

Permanent knowledge base. Nothing here expires.

**Rule:** if it has an expiry date (task, bug, version) → Linear. If it's a decision or context → here.

**Bridge:** Linear issues reference vault notes via `**Vault:** wiki/[folder]/[file].md §section`.

---

## Folders

| Folder | Purpose |
|---|---|
| [`features/`](features/) | One note per feature — why it was built, business rules, edge cases |
| [`decisions/`](decisions/) | ADRs — architecture and product decisions with rationale |
| [`architecture/`](architecture/) | System diagrams (PlantUML) |
| [`domain/`](domain/) | Business domain knowledge — services, plans, pricing, workflows |
| [`people/`](people/) | Stakeholder profiles — contacts, roles, communication preferences |
| [`meetings/`](meetings/) | Dated meeting notes — decisions, action items, open questions |
| [`gaps/`](gaps/) | Open questions pending resolution |
| [`integrations/`](integrations/) | Third-party service docs (Supabase, Resend, Vercel) |
| [`security/`](security/) | Security reviews, auth notes, RLS design |
| [`runbooks/`](runbooks/) | Operational guides — deploy, migrations, incident recovery |
| [`versions/`](versions/) | Version history and release notes |
